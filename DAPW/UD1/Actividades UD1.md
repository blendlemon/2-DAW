# Actividad 1.2 Explica con tus palabras los protocolos ARP, UDP, TCP, TFTP, POP y NTP

- ARP
	- Da IP dinámicas a equipos con direcciones MAC fijas dentro de la mima red LAN
- UDP
	- Manda datagramas conexiones, ofreciendo latencias bajas y tolerancia a la perdida de datos
- TCP
	- Establece una conexion fiable y ordenada entre aplicaciones para el intercambio de datos mediante una red a través de direcciones IP
- TFTP
	- Permite a los clientes obtener y almacenar ficheros en host remotos
- POP
	- Da los emails al cliente desde el servidor de mails
- NTP
	- Sincroniza los relojes del ordenador con los de la red manteniendo los dispositivos sincronizados

# Actividad 1.3 Busca otras tecnología del lado del servidor y cliente
- Lado servidor
	- Grails
		  Un framework de desarrollo web de alta productividad para la plataforma Java, escrito en el lenguaje Groovy. Es conocido por su facilidad para crear aplicaciones web robustas.
	- Play Framework
		  Un framework web para Java y Scala que se enfoca en la escalabilidad y la simplicidad, ofreciendo un enfoque asíncrono y no bloqueante.
	- Computación sin servidor
		  Aunque más una arquitectura, es un enfoque poco común para muchos. El proveedor de la nube aloja y administra los servidores, y la aplicación se ejecuta en respuesta a eventos, lo que permite una gran escalabilidad y costos variables.
	- WebAssembly
		  Un formato de instrucción de bajo nivel que permite ejecutar código compilado de lenguajes como C/C++ directamente en el navegador del cliente, abriendo la puerta a aplicaciones más complejas que antes eran exclusivas del lado del servidor.
- Lado cliente
	- Mithril.js
		  Un micro-framework de JavaScript para la creación de interfaces de usuario modernas y SPA (Single Page Applications). Es ligero y eficiente para aplicaciones que requieren alta interactividad.
	- Vue.js
		  Un framework progresivo que permite la creación de interfaces de usuario avanzadas. Aunque cada vez más popular, para casos específicos puede considerarse una tecnología menos común que sus competidores principales.
	- WebGL
		  Una API de JavaScript para renderizar gráficos 2D y 3D interactivos dentro de un navegador web sin la necesidad de plugins. Permite crear experiencias visuales ricas, como juegos o visualizaciones de datos complejas.
	- Preact
		  Una alternativa ligera a React con la misma API moderna, que permite construir interfaces de usuario rápidamente con un tamaño de archivo mucho menor.

# Diferencia entre web 3.0 y web 4.0
La web 3.0 intenta dar a los usuarios más control sobre sus datos y una web mas inteligente, mientras que la 4.0 busca que la tecnología sea invisible, intuitiva y proactiva, con el objetivo de una conexión sin fisuras entre personas y objetos.
